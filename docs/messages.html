Alerts and Messages

Alerts

This components used to place errors, warnings and informational messages inside content or in dialog boxes.

Basic

Implementation is easy: just add .alert class to any element.

<div class="alert">
    Simple alert message
</div>

It will looks like this:

Simple alert message

Close button

You can add close/dismiss button inside alert container. Its up to developer to provide this functionality, just add .hide class to main .alert element.

<div class="alert">
    <i class="alert__close"></i>
    Simple alert message
</div>
Dismissable alert

Add Icon

You can add any icon along with text message. Use .alert__icon for icon and wrap text into .alert__text to make them align vertically.

<div class="alert">
    <i class="icon icon-checked alert__icon"></i>
    <div class="alert__text">Alert text message</div>
</div>
Alert text message
Can be multiline

Color Options

Additional classes can change color scheme: .alert-success, .alert-danger, .alert-info

Snackbar

Used to dynamically display short notification messages without interrupting user tasks. Example: action completed message, some unobtrusive error messages, not-critical notifications.

Only one message can be displayed at any moment. New message should replace previous one.

Snackbar messages should automatically disappear after specified time period, and can be dismitted by user via clicking or tapping.

See Snackbar component in Google Material Design guidelines.

<div class="snackbar">
    <div class="snackbar__message">Message Text.</div>
</div>

Optional classes .snackbar-success or .snackbar-danger can be assigned to main element to change appearence.

Snackbar message can optionally contain one (and only one!) associated action, as button on the right:

<div class="snackbar">
    <div class="snackbar__actions"><button class="btn btn-link text-uppercase">Action</button></div>
    <div class="snackbar__message">Message with action.</div>
</div>

Note: scripting for snackbar implementation is not part of this library and should be implemented from scratch using chosen framework.
