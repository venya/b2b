Building different page structure

Generic HTML structure

Use following recommended structure for new pages:

<body class="animated">
	<article class="pos pos-payment">
		Main content
	</article>
</body>

Body have no margins and configured to takes up entire available screen. Its colors can be configured through variables.

.animated class should be used on top-most (body) element to allow animations and some heavy UI effects (menu slide-in, animated semi-transparent shadows and masks, dialogs slide-in appearence, dropdown effects, button ripple effects, etc.).
When effects are too slow for some devices, you remove this class programmatically or implement user-accessible per-device settings.

Single Page Applications (SPA)

When building SPA, there should be multiple screens defined in DOM, while only one is currently visible on screen.

Use article container to define each screen. It also takes entire screen, so you can used freely to define any full-screen interfaces with one or multiple columns.

If you need to change background color for some pages, please add new class and make changes athached to it. In the example above we have class="pos pos-payment" to apply both page-group-wide "pos" styles and page-specific "pos-payment" styling.

Multiple Columns

Screen can be presented in multi-column layout. Just place main and aside containers inside article.

<article>
	<main>Main content</main>
	<aside>Additional info</aside>
</article>

Default application-wide columns distribution is 60/40% (configured in _variables.scss) with main content on the left.

.left and .right classes can be assigned to main and aside to change its position:

<article>
	<main class="right">Main content</main>
	<aside class="left">Additional info</aside>
</article>

When custom column width or other layouts needed, assign group/page specific class to article and redefine corresponding main/aside properties in SCSS. Note: multple aside's can be used to make 3-column layouts.
