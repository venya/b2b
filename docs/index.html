Introduction

General concepts behind UI framework

SASS Sources

All components implemented in-house and presented in SASS sources (SCSS syntax format).

Every component or group resides in its own file (e.g. _menu.scss, _buttons.scss, etc). _variables.scss defines globally available variables and settings.

pos.scss used as a main assembly file, which includes most components and page-specific rules, used globally for entire eHopper POS website.

Branching

If you need only subset of all available components, want to create branded, customized or redesigned elements, or use it in side project (e.g. BackOffice, Support Portal, etc) - it is best to create separate main .scss file and @include only needed components, after reconfiguring global variables.

Then link only one compiled .css file to your project.

prelogin.scss is an example of such subset. It only contain elements needed for login, activation and onboarding process - where only small subset of elements used, along with different generic page layout. But it still leverages shared SASS codebase.
