Dialogs as Modal windows

Modals used to represent information in focuses context. User should complete working with dialog content and close it to proceed with rest of the application. That is why semitransparent mask displayed under dialog to dim out screen content.

There are no way to display more than one active dialogs to work with. But dialogs can be nested (dialog can be sown above another dialog, and so on).

<div class="dialog">
	<div class="dialog__content">
		<header class="header">
			<div class="header__icons"><i class="btn icon icon-close dialog__close"></i></div>
			<h1>Dialog Header</h1>
		</header>
		<section>
			This is Dialog content
		</section>
	</div>
</div>
<div class="dialog__mask"></div>

How to Open?

When dialog HTML structure already presented in DOM, the only thing one should to — add .dialog-open class to corresponding .dialog element.

Transparent backdrop (aka Mask) will be automatically animated when any element having .dialog__mask exists in DOM directly after .dialog.

To close dialog - just remove .dialog-open class.

Prevent scrolling

When page content allows scrolling of content (like this one, but not regular POS screens) dialog will scroll with content (this allows to see large dialog content).

To alter this behaviour just put additional .dialog-fixed class to .dialog element.

<div class="dialog dialog-fixed">

Help Panel

Help Panel

Help panel is a special case of using dialog. It implemented the same way, but only has an additional .dialog-help class.

Snackbar

Used to display short notification messages without interrupting user tasks. Example: action completed message, some unobtrusive error messages, not-critical notifications.

Only one message can be displayed at any moment. New message should replace previous one.

Snackbar messages should automatically disappear after specified time period, and can be dismitted by user via clicking or tapping.

See Snackbar component in Google Material Design guidelines.

<div class="snackbar">
    <div class="snackbar__message">Message Text.</div>
</div>

Optional classes .snackbar-success or .snackbar-danger can be assigned to main element to change appearence.

Snackbar message can optionally contain one (and only one!) associated action, as button on the right:

<div class="snackbar">
    <div class="snackbar__actions"><button class="btn btn-link text-uppercase">Action</button></div>
    <div class="snackbar__message">Message with action.</div>
</div>

Note: scripting for snackbar implementation is not part of this library and should be implemented from scratch using chosen framework.
